1 # Naive Bayes
2 This is a c implementation of the Machine Learning algorithm called Naive Bayes Gaussian.
3 It's been adapted from the python version in: [Naive Bayes Classifier from Scratch in Python](http://machinelearningmastery.com/naive-bayes-classifier-scratch-python/).
4 This algorithm was developed to be used in the Embbeded System Module at IFCE to be run in an **ARM9 platform**.
5 
6 This algorithm is divided into two steps:
7 - **Training**, where the dataset will be used to generate the summaries (means and standard deviation) intermediate dataset. This phase happens in a computer.
8 - **Prediction**, where the intermediate dataset + testset will be use to generate the predictions. The prediction phase is the one that runs in the ARM9 platform
9 
10 
11 ## 1) Compiling
12 To get it up and running in a computer, execute the following steps:
13 1. Select a dataset by commenting/uncommenting the datasets define section at the _defines.h_ file;
14 2. Run the _training/main.c_ file to generate the training files;
15  This can also be done by running _make -f makefile_ inside the training folder
16 3. Run the _prediction/main.c_ file to see the classification report (accuracy, recall, precision etc);
17  This can also be done by running _make -f makefile_ inside the prediction folder
18 4. Run your program by typing _./main_ inside the prediction folder
19 
20 
21 To compile the code for the ARM9 plataform:
22 - Repeat the steps 1 and 2 in a PC
23 - Run the command **arm-linux-gcc main.c prediction.c extras.c csv.c -o main -lm** from within the prediction folder in a PC with the ARM compiler
24 - In the ARM platform simply run your program by typing _./main_ from within the prediction folder

34 ## 3) Project structure
35 
36 _The project folder structure is defined as follows:_
37 * **NaiveBayes** - root folder
38  * **prediction** - this folder contains the code used to make the predictions (these are the files that execute the algorithm that runs in the ARM platform)
39  * **testers** - this folder contains the codes for the test code (python with library and original python code)
40  * **training** - this folder contains the code used to generate the summaries and test set
41  * **datasets** - this folder contains all datasets that will be used throughout the execution
42  * **full** - this folder contains the full version of the datasets to be used
43  * **summaries** - this folder contains the summarised knowledge of the classifier (means and standard deviation for each class and attribute)
44  * **test** - this folder contains the test set to be classified
